# Пример 1: Проект "hello_centos", сборка rpm пакета из одного *.py файла

Создание RPM-пакета из Python-скрипта — это отличный способ удобного распространения и установки вашей программы на серверах CentOS, Red Hat (RHEL), AlmaLinux или Rocky Linux.

Ниже представлено подробное пошаговое руководство. Мы создадим простую программу, упакуем её в архив, напишем конфигурационный файл (SPEC) и соберем RPM-пакет.

---

### Шаг 1: Подготовка окружения (Установка утилит)

Сначала нужно установить инструменты для сборки пакетов. Откройте терминал в CentOS и выполните:

```bash
sudo dnf install -y rpm-build rpmdevtools python3
```
*(Если у вас старая версия CentOS 7, используйте `yum` вместо `dnf`).*

Теперь создадим стандартную структуру директорий для сборки RPM в вашей домашней папке. **Никогда не собирайте пакеты от пользователя `root`!**

```bash
rpmdev-setuptree
```

Эта команда создаст скрытую папку `~/rpmbuild/` со следующими подпапками:
*   `BUILD` — временная папка, где происходит сборка.
*   `RPMS` — здесь появятся готовые `.rpm` пакеты.
*   `SOURCES` — сюда кладутся исходники (архивы с кодом).
*   `SPECS` — здесь лежат `.spec` файлы (инструкции по сборке).
*   `SRPMS` — готовые исходные RPM пакеты (Source RPM).

---

### Шаг 2: Создание Python-программы

Давайте напишем простую утилиту командной строки, которая приветствует пользователя и показывает текущее время. Назовем её `hello_centos`.

Создайте папку для исходников нашей версии 1.0:
```bash
mkdir -p ~/hello_centos-1.0
cd ~/hello_centos-1.0
```

Создайте файл `hello_centos.py` (например, с помощью `nano hello_centos.py` или `vi`):

```python
#!/usr/bin/env python3
import sys
import datetime

def main():
    # Получаем имя пользователя из аргументов или используем "User" по умолчанию
    name = sys.argv[1] if len(sys.argv) > 1 else "User"
    now = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    
    print(f"=======================================")
    print(f"  Привет, {name}! Добро пожаловать в CentOS.")
    print(f"  Текущее время: {now}")
    print(f"=======================================")

if __name__ == "__main__":
    main()
```

Сделайте файл исполняемым:
```bash
chmod +x hello_centos.py
```

---

### Шаг 3: Подготовка исходного архива

Сборщик RPM ожидает, что исходный код поставляется в виде сжатого tar-архива (tarball) в папке `SOURCES`.

Выйдите из папки программы и создайте архив:
```bash
cd ~
tar -czvf hello_centos-1.0.tar.gz hello_centos-1.0/
```

Переместите созданный архив в папку `SOURCES` сборщика RPM:
```bash
mv hello_centos-1.0.tar.gz ~/rpmbuild/SOURCES/
```

---

### Шаг 4: Написание SPEC-файла

SPEC-файл — это рецепт, который говорит `rpmbuild`, как распаковать, собрать и установить вашу программу.

Создайте файл `hello_centos.spec` в папке `SPECS`:
```bash
nano ~/rpmbuild/SPECS/hello_centos.spec
```

Скопируйте и вставьте туда следующий текст:

```specfile
Name:           hello_centos
Version:        1.0
Release:        1%{?dist}
Summary:        A simple Python greeting script for CentOS

License:        MIT
Source0:        %{name}-%{version}.tar.gz

BuildArch:      noarch
Requires:       python3

%description
This is a very simple Python command line application 
that prints a greeting and the current time. 
It serves as an example of creating an RPM package.

%prep
# Макрос %setup автоматически распаковывает архив из Source0
%setup -q

%build
# Для простых Python-скриптов здесь ничего компилировать не нужно
# Если бы это был C/C++, здесь был бы ./configure и make

%install
# Удаляем старые следы сборки (хорошая практика)
rm -rf %{buildroot}

# Создаем папку /usr/bin в виртуальном корне сборки
mkdir -p %{buildroot}%{_bindir}

# Копируем наш скрипт в /usr/bin, при этом убираем расширение .py
# чтобы программа вызывалась просто как hello_centos
install -m 0755 hello_centos.py %{buildroot}%{_bindir}/%{name}

%files
# Перечисляем все файлы, которые должны попасть в итоговый RPM пакет
%{_bindir}/%{name}

%changelog
* Fri Mar 06 2026 Admin - 1.0-1
- Initial release of hello_centos package.
```

**Разбор важных параметров SPEC-файла:**
*   `BuildArch: noarch` — означает "no architecture". Так как Python — интерпретируемый язык, нам не нужно собирать разные пакеты для x86_64 или ARM. Один пакет будет работать везде.
*   `Requires: python3` — указывает менеджеру пакетов (dnf/yum), что при установке этого RPM нужно автоматически установить `python3`, если его нет.
*   `%{buildroot}` — временная директория, имитирующая корневую файловую систему (`/`) во время установки.
*   `%{_bindir}` — системный макрос, который превращается в `/usr/bin`.

---

### Шаг 5: Сборка RPM-пакета

Теперь всё готово для сборки. Запустите команду:

```bash
rpmbuild -ba ~/rpmbuild/SPECS/hello_centos.spec
```
*(Флаг `-ba` означает "build all" — собрать и бинарный RPM, и исходный SRPM).*

Вы увидите много текста в терминале. Если всё прошло успешно, в конце будут строки вида:
```text
+ exit 0
```

Проверим, создался ли пакет:
```bash
ls -l ~/rpmbuild/RPMS/noarch/
```
Вы должны увидеть файл `hello_centos-1.0-1.el8.noarch.rpm` (или `el9`, в зависимости от версии вашей ОС).

---

### Шаг 6: Установка и тестирование пакета

Теперь мы можем установить наш пакет в систему так же, как любую другую программу.

```bash
sudo dnf localinstall ~/rpmbuild/RPMS/noarch/hello_centos-1.0-1.*.rpm -y
```

Проверим работу программы! Так как мы положили скрипт в `/usr/bin/hello_centos`, он доступен из любой точки системы.

В терминале введите:
```bash
hello_centos
```
Вывод:
```text
=======================================
  Привет, User! Добро пожаловать в CentOS.
  Текущее время: 2026-03-06 15:30:45
=======================================
```

Или передайте аргумент:
```bash
hello_centos Алексей
```
Вывод:
```text
=======================================
  Привет, Алексей! Добро пожаловать в CentOS.
...
```

---

### Шаг 7: Удаление пакета (Уборка)

Чтобы убедиться, что менеджер пакетов полностью контролирует вашу программу, удалите её:

```bash
sudo dnf remove hello_centos -y
```
Попробуйте снова выполнить `hello_centos` — система скажет, что команда не найдена. Пакет удален корректно.

---

### Резюме для более сложных Python-программ
Если ваша программа состоит из множества файлов или использует сторонние библиотеки (`requests`, `flask` и т.д.), подход немного меняется:
1.  Обычно используют инструменты сборки Python (например, `setup.py` или `pyproject.toml`).
2.  В секции `%build` SPEC-файла вызывается команда сборки (например, `python3 setup.py build`).
3.  В секции `%install` вызывается `python3 setup.py install --root %{buildroot}`.
4. В секции `Requires` указываются системные RPM-пакеты Python-зависимостей (например, `python3-requests`).
